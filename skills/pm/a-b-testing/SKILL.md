# A/B Testing Skill

## 触发条件
- 需要数据驱动方式验证产品改进
- 多个方案难以决策
- 功能上线后需要衡量效果

## 执行步骤
1. 明确实验目标和对照组
2. 计算所需样本量和实验时长
3. 设计实验变量和分流方案
4. 启动实验并监控数据
5. 分析统计显著性并做出决策

## 最佳实践
- 每次只测试一个变量
- 提前约定决策标准
- 避免过早得出结论
- 记录所有实验和结论

## 常见陷阱
- 样本量不足导致统计不显著
- 实验组之间相互干扰
- 选择性报告有利结果(p-hacking)
