# User Story Mapping Skill

## 触发条件
- 需要为新产品或功能制定开发计划
- 需要将产品路线图转化为具体的开发任务
- 需要与团队对齐产品功能优先级

## 执行步骤
1. 识别用户角色和核心活动
2. 按照用户旅程梳理主干活动
3. 在每个活动中拆解用户任务
4. 按优先级划分发布计划
5. 标注依赖关系和风险

## 最佳实践
- 从用户视角出发，而非系统功能角度
- 使用便利贴或在线工具进行协作
- 每个发布版本聚焦可交付的用户价值
- 定期回顾和调整故事地图

## 常见陷阱
- 贪多求全，第一版试图包含所有功能
- 故事颗粒度不一致
- 忽略技术依赖和基础设施需求
