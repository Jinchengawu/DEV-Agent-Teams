# Roadmap Planning Skill

## 触发条件
- 需要向干系人展示产品规划
- 制定季度/年度产品计划
- 协调多个团队的开发节奏

## 执行步骤
1. 收集需求池和产品目标
2. 按主题 (Theme) 组织功能
3. 确定各主题的优先级和时间范围
4. 标注关键里程碑和依赖
5. 制作路线图并持续更新

## 最佳实践
- 路线图聚焦"为什么"而非"做什么"
- 使用 Now/Next/Later 而非精确日期
- 区分承诺项和探索项
- 每季度审视和调整路线图

## 常见陷阱
- 路线图变成详细的功能清单
- 过于乐观的时间估算
- 对变更缺乏灵活性
