# Prototyping Skill

## 触发条件
- 需要快速验证产品概念
- 向团队或干系人展示功能方案
- 进行可用性测试前的原型准备

## 执行步骤
1. 确定原型目标和保真度级别
2. 绘制核心流程的线框图
3. 添加关键交互和页面状态
4. 与团队进行评审和迭代
5. 准备用于用户测试的原型

## 最佳实践
- 从纸面原型开始，快速迭代
- 先用低保真验证流程，再用高保真验证细节
- 覆盖正常流程和异常状态
- 标注交互说明和边界条件

## 常见陷阱
- 过早投入高保真原型
- 原型只覆盖正常路径
- 在原型中过度追求美观而非可用性
