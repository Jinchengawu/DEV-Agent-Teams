# User Research Skill

## 触发条件
- 需要深入了解用户痛点和需求
- 产品功能上线前需要验证方案
- 产品使用数据异常需要归因

## 执行步骤
1. 明确研究目标和待验证假设
2. 选择合适的研究方法（访谈/问卷/可用性测试）
3. 招募目标用户
4. 执行研究并记录观察
5. 分析发现并输出研究报告

## 最佳实践
- 5个用户即可发现85%的可用性问题
- 避免引导性问题
- 观察用户行为而非只听用户说
- 将研究结果与团队共享

## 常见陷阱
- 只访问容易找到的用户（朋友/同事）
- 向用户问他们想要什么功能
- 忽视沉默的大多数用户
