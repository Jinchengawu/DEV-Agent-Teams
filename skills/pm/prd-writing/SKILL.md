# PRD Writing Skill

## 触发条件
- 用户需要撰写产品需求文档
- 需要明确功能规格和验收标准
- 需要向开发团队传达产品需求

## 执行步骤
1. 明确产品目标和用户价值
2. 定义功能范围和非功能需求
3. 编写用户场景和用例
4. 制定验收标准和成功指标
5. 组织文档结构并输出

## 最佳实践
- 使用用户故事格式：作为<角色>，我想要<功能>，以便<价值>
- 每个功能需明确优先级 (P0/P1/P2)
- 包含交互原型或线框图的文字描述
- 明确非功能性需求（性能、安全、可用性）

## 常见陷阱
- 需求过于模糊，开发团队无法执行
- 过度指定实现细节，限制技术方案选择
- 缺少边界条件和非正常流程的处理

## 示例代码
```markdown
# PRD: 用户登录功能
## 背景
当前系统缺少安全的用户认证机制
## 目标
实现邮箱+密码登录，支持记住密码和找回密码
## 用户故事
- 作为用户，我希望能用邮箱注册账号
- 作为用户，我希望能用邮箱+密码登录
## 验收标准
- 登录接口响应时间 < 500ms
- 密码至少8位，包含大小写字母和数字
```
