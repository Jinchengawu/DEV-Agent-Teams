# Requirements Analysis Skill

## 触发条件
- 收到模糊的业务需求需要澄清
- 多个需求冲突需要优先级排序
- 需要从用户反馈中提取有效需求

## 执行步骤
1. 收集需求来源（用户反馈、业务目标、竞品分析）
2. 分类和归纳需求
3. 评估需求价值和实现成本
4. 使用 RICE/MoSCoW 等方法排定优先级
5. 输出需求分析报告

## 最佳实践
- 区分用户需求与产品需求
- 用数据支撑需求优先级决策
- 建立需求追溯矩阵
- 定期与干系人对齐需求理解

## 常见陷阱
- 将解决方案当作需求
- 忽视非功能性需求
- 需求漫延(Scope Creep)
